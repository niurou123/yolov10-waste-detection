# YOLOv10 垃圾分类检测 — 前端设计规格

**日期：** 2026-06-08
**项目：** 基于 YOLOv10 的垃圾识别目标检测与检测研究
**作者：** 刘汪洋 (235703240)

---

## 1. 概述

为现有 YOLOv10 垃圾分类检测模型（4 类别，best.pt）构建一个 Web 前端，
用于成果展示和实际检测使用。

### 当前状态

- 已有 Gradio demo (`app.py`)，功能有限，界面简陋
- 已有训练好的模型：`runs/detect/train5/weights/best.pt`
- 检测类别：可回收垃圾、有害垃圾、厨余垃圾、其他垃圾

### 目标

构建一个独立 Web 应用，提供：
- 单张图片上传 + 实时检测 + 结果可视化
- 批量图片上传 + 批量检测 + 结果画廊
- 结果下载（标注图片、统计报告）

---

## 2. 设计决策

| 决策项 | 选择 | 依据 |
|--------|------|------|
| 前端形态 | 独立 Web 应用 | 答辩展示需要精美 UI，Gradio 不够 |
| 视觉风格 | TechDark（科技暗黑） | 强调 AI 技术感，深色背景 + 蓝紫荧光 |
| 技术方案 | 单 HTML + FastAPI | 项目规模适中，React 过度工程化 |
| 部署模式 | 本地运行（localhost） | 学生项目，非生产环境 |

---

## 3. 系统架构

```
浏览器 (SPA)
    │  HTTP REST API
    ▼
FastAPI 后端 (:8000)
    │  ultralytics
    ▼
YOLOv10 Model (best.pt)
    └── 4 类别：可回收 / 有害 / 厨余 / 其他
```

### 前端（单 HTML 文件）

- **技术栈：** 纯 HTML5 + Tailwind CSS CDN + 原生 JavaScript
- **页面：** 单页面，通过 Tab 切换"单张检测"和"批量检测"
- **无需构建：** 直接在浏览器中打开

### 后端（FastAPI）

- **框架：** FastAPI + uvicorn
- **模型：** ultralytics YOLOv10，加载 `best.pt`
- **API 端点：**

| 端点 | 方法 | 输入 | 输出 |
|------|------|------|------|
| `/api/detect` | POST | 单张图片 (multipart/form-data) | 标注图片 (base64) + 检测结果 JSON |
| `/api/batch-detect` | POST | 多张图片 (multipart/form-data) | ZIP 包（标注图片） + 汇总统计 JSON |
| `/api/health` | GET | — | 模型状态 + 类别信息 |

## 5. API 接口规格

### POST /api/detect

```
请求：multipart/form-data
  - image: File (jpg/png/bmp)
  - conf: float (可选, 默认 0.25)
  - imgsz: int (可选, 默认 640)

响应：application/json
{
  "success": true,
  "image_base64": "data:image/jpeg;base64,/9j/...",  // 标注后的图片
  "detections": [
    {
      "class_id": 0,
      "class_name": "recyclable waste",
      "class_name_zh": "可回收垃圾",
      "confidence": 0.94,
      "bbox": [x1, y1, x2, y2]
    }
  ],
  "count": 3,
  "inference_time_ms": 830
}
```

### POST /api/batch-detect

```
请求：multipart/form-data
  - images: File[] (多张图片)
  - conf: float (可选, 默认 0.25)

响应：application/json
{
  "success": true,
  "total_images": 12,
  "total_detections": 45,
  "results": [
    {
      "filename": "img_001.jpg",
      "image_base64": "...",
      "detections": [...],
      "count": 3
    }
  ],
  "summary": {
    "recyclable waste": 12,
    "hazardous waste": 8,
    "kitchen waste": 15,
    "other waste": 10
  }
}
```

## 7. 交互流程

### 单张检测
```
用户打开 index.html → 上传图片（拖拽/点击）
  → 点击"开始检测" → 前端 POST /api/detect
  → 后端加载模型 → 推理 → 标注 bounding box
  → 返回标注图 + JSON → 前端渲染结果
```

### 批量检测
```
用户切换到"批量检测" Tab → 上传多张图片
  → 点击"开始检测" → 前端逐张 POST /api/detect（显示进度条）
  → 全部完成 → 画廊展示 + 下载按钮
```

## 8. 范围边界

**包含：**
- 单张图片检测 + 结果显示
- 批量图片检测 + 结果画廊
- 检测参数调节（置信度、图片尺寸）
- 结果下载（标注图片）
- 检测统计汇总

**不包含：**
- 视频检测（后续迭代）
- 实时摄像头检测（后续迭代）
- 用户登录/权限
- 检测历史持久化
- 模型切换（固定使用 best.pt）
- 部署到公网

---

## 9. 技术约束

- **运行环境：** Windows 10/11，Python 3.9+
- **模型路径：** 硬编码 `runs/detect/train5/weights/best.pt`
- **GPU：** 后端优先使用 CUDA（`device=0`），无 GPU 时回退 CPU
- **CORS：** FastAPI 需配置允许跨域（前端 file:// 协议打开）
- **文件大小：** 单张图片限制 10MB，批量一次最多 50 张
